# Admin Panel Balance Storage

Цель: баланс админ-панели должен храниться отдельно, по месяцам и по строкам, чтобы данные не удалялись при смене назначений анкет.

## Что важно сохранить

Баланс нельзя хранить только как текущую сумму на экране. Нужно хранить сырые строки, из которых сумма считается.

Правильная модель:

- сырые финансовые строки сохраняются навсегда;
- таблица месяца строится из этих строк;
- если анкету сняли, старые строки остаются;
- если админа удалили, старые строки остаются;
- если поменяли имя анкеты, старые суммы остаются привязаны к `profileId`.

## Рекомендуемая будущая структура

```text
runtime-data/
  admin-panel/
    ledger/
      2026-06.json
      2026-07.json
    snapshots/
      2026-06.json
      2026-07.json
    cell-colors/
      2026-06.json
      2026-07.json
    cell-comments/
      2026-06.json
      2026-07.json
    refresh-state.json
```

## `ledger/YYYY-MM.json`

Главный источник правды. Каждая строка из агентской панели сохраняется отдельно.

Пример:

```json
{
  "unique-row-key": {
    "id": "unique-row-key",
    "date": "2026-06-24T12:30:00.000Z",
    "businessDate": "2026-06-24",
    "profileId": "17838562",
    "profileName": "Tetyana",
    "operatorId": "5487e124-413a-42ef-8545-921b96bec8f0",
    "operatorName": "Vadym (TOP)",
    "amount": 12.5,
    "type": "letter",
    "source": "agency-bonuses",
    "raw": {}
  }
}
```

Почему так:

- если анкета снята завтра, строка за сегодня остается за старым оператором;
- если таблицу пересобрали, сумма не пропадает;
- если имя изменилось, ID остается тем же.

## `snapshots/YYYY-MM.json`

Снимок готовой таблицы. Это кеш, а не главный источник.

Можно хранить:

- строки операторов;
- строки анкет;
- дневные суммы;
- gifts;
- итог месяца;
- дату генерации.

Если snapshot сломался, его можно пересобрать из `ledger`.

## `cell-colors/YYYY-MM.json`

Цвета ячеек:

```json
{
  "operator:5487e124:2026-06-24": "#fff2b8",
  "profile:5487e124:17838562:2026-06-24": "#e8f5e9"
}
```

Цвета не должны храниться внутри строк баланса, чтобы обновление баланса не стирало ручные отметки.

## `cell-comments/YYYY-MM.json`

Комментарии к ячейкам и строкам:

```json
{
  "profile-name:5487e124:17838562": "comment text",
  "profile:5487e124:17838562:2026-06-24": "comment text"
}
```

Комментарии тоже отдельно от ledger.

## `refresh-state.json`

Техническое состояние обновления:

- последний refresh;
- последняя успешная страница;
- ошибки;
- кто запускал обновление.

## Главное правило

Баланс хранится не "у админа" и не "у анкеты", а в общем ledger с обязательными ссылками:

- `profileId`;
- `operatorId`;
- `businessDate`;
- `amount`;
- `source`.

Так админ-панель может показать прошлый месяц даже если:

- анкету сняли;
- админа удалили;
- профиль переименовали;
- текущие назначения изменились.

## Что нельзя делать

- Нельзя удалять ledger при снятии анкеты.
- Нельзя пересчитывать прошлые строки по текущему владельцу анкеты.
- Нельзя хранить только готовую таблицу без сырых строк.
- Нельзя смешивать цвета/комментарии с финансовыми строками.
