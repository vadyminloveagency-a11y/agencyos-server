# CRM Data Map

Эта папка описывает, какие данные есть в CRM, где они сейчас лежат и какие правила нельзя ломать при доработках.

Сейчас живые данные приложения хранятся в одном файле `data.json`. Это нормально для локальной CRM, но опасно тем, что разные разделы используют один общий объект. Поэтому эта папка служит картой границ: если правим один раздел, не трогаем чужие поля и алгоритмы.

Важно: эта папка не переносит живые данные. Она фиксирует правила и границы. Реальный перенос из `data.json` в отдельные файлы нужно делать отдельным шагом с миграцией и проверками.

## Разделы

- `auth/` - логины, пароли, сессии, роли.
- `profiles-assignments/` - анкеты, владельцы, назначение админам/операторам, история назначений.
- `admin-panel/` - таблица админ-панели, балансы, цвета, комментарии, месячная история.
- `inbox/` - входящие письма, диалоги, статусы прочитано/не отвечено.
- `sent/` - отправленные письма и история исходящих.
- `mail-media/` - фото и видео для писем, Dream gallery id, вложения.
- `dashboard/` - основной экран, списки мужчин, избранное, заметки, статусы.
- `stats-salary/` - статистика, зарплаты, проценты, дневные/месячные итоги.
- `settings/` - настройки CRM, Google Drive, переводчик, агентские доступы.
- `external-services/` - Dream Singles, Agency bonuses, Google/DeepL и другие внешние источники.

## Главное правило для будущих правок

Перед изменением кода нужно определить раздел из этой карты и менять только его. Если задача про `admin-panel`, нельзя менять `inbox`, `sent`, `mail-media` без отдельного согласия. Если задача про письма, нельзя менять назначения анкет и таблицу.

Живой `data.json` нельзя править руками без прямого разрешения владельца CRM.
