# Dashboard Data

Раздел отвечает за основной экран CRM: мужчины, избранное, заметки, статусы, фильтры и быстрые действия.

## Где хранится сейчас

В `data.json` внутри анкеты:

- `profiles[profileId].men` - основной список мужчин.
- `profiles[profileId].otherMen` - дополнительные/временные мужчины.
- части данных Inbox могут отображаться на Dashboard, но владельцем данных остается Inbox.

Типичные поля мужчины:

- `id`, `name`, `age`, `country`, `city`;
- `photoUrl`, `profileLink`;
- `favorite`, `siteFavorite`, `pinned`, `ignored`;
- `note`, `status`;
- `onlineNow`, `lastActivityText`, `onlineCheckedAt`.

## Правила Dashboard

- Dashboard может показывать данные Inbox/онлайна, но не должен менять алгоритм писем без отдельной задачи.
- Статусы и заметки мужчины не должны ломать Inbox conversation.
- Избранное на сайте и локальное избранное нужно различать.

## Что нельзя трогать без отдельной задачи

- Admin Panel;
- пароли;
- назначения анкет;
- отправку писем;
- структуру `workspaceInbox`.

## Проверка после изменений

- список мужчин открывается;
- поиск/фильтры работают;
- заметки сохраняются;
- избранное не теряется;
- онлайн-статус не ломает Inbox.
