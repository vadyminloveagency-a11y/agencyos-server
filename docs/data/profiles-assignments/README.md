# Profiles And Assignments Data

Раздел отвечает за анкеты девушек и правила назначения директором, администратором и оператором.

## Где хранится сейчас

В `data.json`:

- `profiles` - анкеты.
- `users[].profileIds` - текущие активные анкеты пользователя.
- `profiles[].ownerAdminId` - владелец/администратор анкеты, если поле используется.
- `assignmentHistory` - история назначений и снятий анкет.

Будущая безопасная структура "одна анкета = одна папка" описана в `PER_PROFILE_STORAGE.md`.

## Основные поля анкеты

- `id` - ID анкеты Dream Singles.
- `name` - имя анкеты.
- `active` - существует ли анкета в CRM.
- `photoUrl` - фото.
- `credentials` - зашифрованные логин/пароль анкеты Dream.
- `men`, `otherMen` - мужчины и рабочие списки.
- `workspaceInbox` - сохраненные входящие/диалоги.
- `workspaceMediaGallery` - фото/видео для писем.

## Правила назначения

Эти правила нужно уточнять перед изменениями. Зафиксированная граница:

- директорское назначение анкеты администратору не должно случайно менять Inbox, Sent, Media или Dashboard;
- снятие анкеты должно закрывать историю в `assignmentHistory`;
- если анкета участвовала в месяце, админ-панель может показывать ее как неактивную историческую строку;
- `profileIds` отражает текущий активный доступ, а `assignmentHistory` отражает историю.

## Что нельзя трогать без отдельной задачи

- `assignmentHistory`;
- `users[].profileIds`;
- `profiles[].ownerAdminId`;
- `profiles[].active`;
- удаление анкеты;
- ручную правку `data.json`.

## Проверка после изменений

- директор назначает анкету администратору;
- администратор видит только разрешенные анкеты;
- снятая анкета исчезает из активного выбора, но не теряется из истории месяца;
- таблица не меняет правила Inbox/Sent/Media.
