# Auth Data

Раздел отвечает за вход в CRM, пользователей, роли и сессии.

## Где хранится сейчас

В `data.json`:

- `users` - все пользователи CRM.
- `sessions` - активные сессии входа.

В корне проекта:

- `.credential-key` - ключ для шифрования внешних логинов/паролей анкет и агентских доступов.

## Как хранятся логины и пароли пользователей CRM

У каждого пользователя в `users`:

- `username` - логин.
- `passwordSalt` - соль.
- `passwordHash` - хеш пароля.
- `role` - `director`, `admin` или `operator`.
- `active` - включен ли пользователь.

Пароль пользователя CRM не хранится обычным текстом. При входе сравнивается хеш.

## Как хранятся сессии

В `sessions` хранится хеш токена сессии:

- ключ объекта - хеш cookie `crm_session`.
- `userId` - пользователь.
- `expiresAt` - срок жизни сессии.

Сам токен уходит в браузер как `HttpOnly` cookie.

## Что нельзя трогать без отдельной задачи

- алгоритм `hashPassword` / `passwordMatches`;
- формат `sessions`;
- cookie `crm_session`;
- роли пользователей;
- удаление/деактивацию пользователей.

## Проверка после изменений

- директор может войти;
- администратор может войти;
- оператор может войти, если такая роль используется;
- после logout сессия удаляется;
- пароль не появляется в ответах API.
